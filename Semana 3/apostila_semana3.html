1. Conceitos de Linguagem de Programação

Definição

Uma linguagem de programação é um conjunto de regras, símbolos e palavras-chave que permitem dar instruções ao computador de forma estruturada e compreensível.

Níveis de Linguagem

  • Linguagens de Baixo Nível: Mais próximas do hardware, como Assembly. Difíceis de aprender, mas oferecem controle máximo do sistema.
  • Linguagens de Alto Nível: Mais próximas da linguagem humana, como PHP e JavaScript. Mais fáceis de aprender e manter.

Modelos de Processamento

Tipo Descrição Exemplo
Compilada Traduzida integralmente antes da execução (build) C, C++, Java
Interpretada Traduzida linha a linha em tempo real (runtime) JavaScript, Python
JIT Combinação das duas: compilação durante execução JavaScript moderno (V8)

2. Fundamentos do JavaScript

Papel na Web

JavaScript é uma linguagem de frontend que roda no navegador do cliente, responsável por trazer interatividade às páginas web. Também pode rodar no servidor através do Node.js.

Java vs JavaScript

Esclarecimento importante: Java e JavaScript são linguagens completamente diferentes. O nome foi uma escolha de marketing em 1995 quando JavaScript foi criado. As sintaxes, paradigmas e aplicações são distintos. Java é compilada e roda na JVM; JavaScript é interpretada e roda em navegadores.

Ambiente de Teste: Console do Navegador

Uma das vantagens do JavaScript é a possibilidade de testar código rapidamente:

  • Pressione F12 para abrir as ferramentas do desenvolvedor
  • Acesse a aba Console
  • Digite e execute código JavaScript imediatamente
  • Sem necessidade de criar arquivos ou fazer build

3. Sintaxe e Estruturas de Dados

Variáveis e Constantes

No JavaScript moderno (ES6+), existem duas formas principais de declarar dados:

  • let: Para variáveis que mudam durante a execução
  • const: Para valores constantes (imutáveis)
  • var: Forma ultrapassada, evitar em código novo

Boas Práticas de Nomeação

  • CamelCase: nomeDeVariavel (primeira palavra minúscula, próximas com primeira letra maiúscula)
  • Case Sensitive: JavaScript diferencia MAIÚSCULAS de minúsculas (numero ≠ Numero)
  • Clareza: Nomes descritivos facilitam leitura: contador é melhor que c

Tipos de Dados Primitivos

Tipo Descrição Exemplo
number Inteiros e decimais (sem distinção) 42, 3.14, -7
string Texto encapsulado em aspas "Olá", 'Mundo', `Template`
boolean Verdadeiro ou falso true, false
undefined Valor não definido let x; (x é undefined)
null Ausência proposital de valor let x = null;

Arrays (Vetores)

Arrays são coleções ordenadas de dados. Importante: a indexação sempre começa em zero.

Exemplo: const numeros = [1, 2, 3, 4, 5]
numeros[0] = 1 (primeiro elemento)
numeros[4] = 5 (último elemento)
numeros.length = 5 (propriedade que retorna o tamanho)

4. Controle de Fluxo e Funções

Estruturas de Decisão

Permitem executar diferentes blocos de código baseado em condições:

  • if: Executa se a condição for verdadeira
  • else: Executa se a condição for falsa
  • else if: Múltiplas condições
  • Operador Ternário: (condição ? true : false) — forma simplificada em uma linha

Estruturas de Repetição (Loops)

Loop Uso Características
while Repetir enquanto condição for verdadeira Pode não executar nenhuma vez
do...while Repetir até condição ficar falsa Executa pelo menos uma vez
for Repetir um número fixo de vezes Ideal para contadores e arrays

Funções

Blocos de código reutilizáveis que podem receber parâmetros (entradas) e devolver um return (saída).

  • Encapsulam lógica em unidades independentes
  • Facilitam reutilização e organização do código
  • Melhoram legibilidade e manutenibilidade

ANÁLISE TÉCNICA DO CÓDIGO

Estrutura e Organização

O código dos exercícios foi estruturado seguindo padrões profissionais de desenvolvimento web, separando responsabilidades e facilitando a manutenção. Cada exercício demonstra um conceito específico de programação imperativa.

HTML — Estrutura Semântica

O documento HTML utiliza elementos semânticos apropriados:

  • <header>: Cabeçalho da página com informações contextuais
  • <section>: Agrupamento lógico de conteúdo relacionado
  • <button>: Elemento interativo que dispara eventos
  • <pre>: Preserva formatação de código (espaços e indentação)

CSS — Design System Consistente

A folha de estilos implementa um design system coerente:

  • Variáveis de Cor: Paleta consistente com azul (#1a237e), vermelho (#e50914), verde (#6a9955)
  • Flexbox: Sistema de layout moderno para alinhamento e distribuição de espaço
  • Responsive: Adaptável a diferentes tamanhos de tela (mobile-first)
  • Acessibilidade: Contraste adequado, tamanhos legíveis (rem em vez de px)
  • Transições: Feedback visual suave ao interagir com botões

JavaScript — Lógica Modular

O código JavaScript demonstra boas práticas de programação:

Separação de Responsabilidades

Cada função tem um único propósito claro:

  • contarAteNumero() — realiza a lógica de contagem
  • somarNumeros() — calcula soma sequencial
  • lerNumeroValido() — valida entrada do usuário
  • gerarTabuada() — gera tabuada formatada

Reutilização e DRY (Don't Repeat Yourself)

Funções são chamadas múltiplas vezes sem repetir código. Por exemplo, a função exibirSaida() é usada por todos os botões:

Padrão DRY
// ❌ Ruim — Código repetido
function rodarExercicio1() {
  const saida = document.getElementById("saida1");
  saida.textContent = contarAteNumero();
  saida.className = "saida visivel";
}

// ✓ Bom — Função reutilizável
function exibirSaida(id, texto) {
  const saida = document.getElementById(id);
  saida.textContent = texto;
  saida.className = "saida visivel";
}

Validação de Entrada

Todos os exercícios validam a entrada do usuário antes de processar:

  • isNaN() — verifica se o valor é um número válido
  • Verificações de intervalo — garante que valores estão dentro do esperado
  • Mensagens de erro — feedback claro ao usuário

Tratamento de Erros

Uso de try...catch para capturar exceções (como quando usuário cancela prompt):

Tratamento Seguro
function rodarExercicio1() {
  try {
    exibirSaida("saida1", contarAteNumero());
  } catch (e) {
    // Operação cancelada pelo usuário
    exibirSaida("saida1", "Operação cancelada.", true);
  }
}

Técnicas Avançadas Utilizadas

Template Literals (Crase)

Uso de crases permite interpolar variáveis dentro de strings:

Template Literals
// ❌ Concatenação tradicional (verbosa)
"A soma de 1 até " + numero + " é: " + soma

// ✓ Template literal (legível)
`A soma de 1 até ${numero} é: ${soma}`

Operadores Aritméticos Compostos

Usando += e ++ para código mais limpo:

  • soma += i equivale a soma = soma + i (mais conciso)
  • contador++ equivale a contador = contador + 1 (menos caracteres)

Operadores Lógicos

Combinar múltiplas condições com lógica booleana:

Operador Símbolo Descrição
AND (E) && Verdadeiro APENAS SE ambas as condições forem verdadeiras
OR (OU) || Verdadeiro se QUALQUER UM das condições for verdadeira
NOT (NÃO) ! Inverte o resultado (verdadeiro vira falso e vice-versa)

Manipulação do DOM (Document Object Model)

JavaScript interage com HTML através do DOM:

  • document: Representa a página HTML inteira
  • getElementById(): Busca elemento por ID
  • .textContent: Modifica o texto dentro do elemento
  • .className: Define as classes CSS do elemento

Padrões de Design Aplicados

Single Responsibility Principle (SRP)

Cada função tem uma única responsabilidade bem definida. Facilita testes e manutenção:

Exemplo: A função exibirSaida() é responsável APENAS por atualizar a interface. A lógica de cálculo fica em funções separadas como somarNumeros().

DRY — Don't Repeat Yourself

Código comum é extraído em funções reutilizáveis. Mudanças precisam ser feitas em um único lugar.

Progressive Enhancement

O HTML funciona sozinho (estrutura), CSS melhora a apresentação, JavaScript adiciona interatividade. Se JavaScript falhar, a página ainda é estruturada.

Fluxo de Execução Típico

  1. Usuário clica no botão "Executar"
  2. Função rodarExercicio() é invocada
  3. Dentro dela, a função do exercício é chamada (ex: somarNumeros())
  4. Se prompt() retornar null (cancelado), cai no catch
  5. Resultado é passado para exibirSaida()
  6. exibirSaida() atualiza o DOM com o resultado
  7. CSS faz a
    com classe "visivel" aparecer

TAREFA PRÁTICA DA SEMANA 3

A atividade prática consiste em entregar quatro exercícios que exercitam a lógica de programação, o pensamento algorítmico e a codificação em JavaScript:

Exercício 1: Contar até um Número

Objetivo

Solicitar um número ao usuário e contar de 1 até ele usando o laço while.

Conceitos Aplicados

  • Laço while (enquanto)
  • Operador de incremento (++)
  • Variáveis de controle
  • Acumulação de strings

Exercício 2: Soma de Números Sequenciais

Objetivo

Solicitar um número e somar todos os inteiros de 1 até ele usando for.

Conceitos Aplicados

  • Laço for (para)
  • Variável acumuladora (soma)
  • Operador composto (+=)
  • Padrão de contagem com passo fixo

Exercício 3: Validação com Função

Objetivo

Solicitar um número entre 1 e 60. Enquanto inválido, pedir novamente. Obrigatoriamente usando função.

Conceitos Aplicados

  • Encapsulamento em funções
  • Validação de entrada
  • Laço while com múltiplas condições
  • Operadores lógicos (||, &&)
  • Feedback ao usuário

Exercício 4: Desafio — Tabuada

Objetivo

Criar a tabuada completa de um número (1 a 10) usando for. Formatação visual padronizada.

Conceitos Aplicados

  • Laço for com contagem fixa
  • Operações aritméticas (*)
  • Formatação de strings
  • Método padStart() para alinhamento
  • Acumulação de múltiplas linhas

Metodologia da Tríade de Raciocínio

Para cada exercício, apresentar os três níveis de abstração:

  1. Linguagem Natural: Explicar em português claro, passo a passo, como o algoritmo funciona
  2. Pseudocódigo: Escrever a lógica usando estruturas algorítmicas (não código real)
  3. Código JavaScript: Implementação real e funcional em JavaScript
Objetivo Pedagógico: A tríade trabalha diferentes níveis cognitivos. Linguagem natural desenvolve clareza de pensamento. Pseudocódigo desenvolve estrutura lógica. Código real desenvolve habilidade técnica.

Critérios de Avaliação

  • Funcionalidade: Código executa e produz resultado correto
  • Legibilidade: Variáveis bem nomeadas, indentação correta
  • Documentação: Comentários explicam a lógica
  • Validação: Entrada do usuário é tratada adequadamente
  • Pensamento Lógico: Demonstra compreensão de estruturas de controle

Desenvolvimento e Aprendizado

Boas Práticas de Programação

Padrões Essenciais

  • Nomeação Significativa: Nomes refletem propósito (contador, soma, resultado)
  • DRY: Não Repita Código — extraia em funções
  • KISS: Keep It Simple, Stupid — solução mais direta é melhor
  • Validação: Sempre validar entrada do usuário
  • Comentários: Explicar o "por quê", não apenas o "o quê"

Estrutura de Código Recomendada

Padrão Profissional
// 1. Declarar variáveis e constantes
const numero = parseInt(prompt(...));

// 2. Validar entrada
if (isNaN(numero)) {
  return "Erro: entrada inválida";
}

// 3. Processar lógica
let resultado = 0;
for (let i = 1; i <= numero; i++) {
  resultado += i;
}

// 4. Retornar resultado
return resultado;
