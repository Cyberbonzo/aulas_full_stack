RESUMO DA AULA 4

A Aula 4 da mentoria foi dedicada à JavaScript e Manipulação do DOM (Document Object Model), focando em como o JavaScript "dá vida" às páginas estáticas criadas com HTML e CSS.

1. O que é o DOM?

Conceito Fundamental

O DOM é a representação do HTML como uma árvore de objetos. Cada elemento da página (tags, atributos, textos) é um "nó" nessa árvore, começando pela raiz document.

Estrutura em Árvore

Exemplo:
document
└── html
    ├── head
    │    └── title
    └── body
        ├── header
        └── div (class="container")

Boas Práticas

  • Script no final do <body>: Garante que todos os elementos HTML já existem antes de JavaScript tentar acessá-los
  • Não manipular antes de carregamento: Usar eventos como DOMContentLoaded se script estiver no <head>

2. Métodos de Acesso aos Elementos

JavaScript fornece diversas formas de encontrar elementos na página:

Método Sintaxe Retorna
getElementById getElementById('id') Um elemento (único ou null)
getElementsByClassName getElementsByClassName('classe') HTMLCollection (múltiplos)
querySelector querySelector('#id' ou '.classe') Primeiro elemento ou null
querySelectorAll querySelectorAll('.classe') NodeList (múltiplos)

querySelector vs getElementById

querySelector é mais moderno:
✗ getElementById('meuId')
✓ querySelector('#meuId')

✗ getElementsByClassName('ativa')
✓ querySelector('.ativa') ou querySelectorAll('.ativa')

3. Manipulação de Conteúdo e Estilo

Conteúdo (Propriedades de Texto)

Propriedade Descrição Segurança
innerHTML Altera HTML interno (permite tags) ⚠️ Risco XSS se dados são do usuário
textContent Altera apenas texto (mais seguro) ✓ Seguro contra injeção
value Lê/modifica valor de inputs ✓ Seguro (campo de form)

Estilo (Propriedade .style)

Modificar CSS via JavaScript usando a propriedade .style:

  • Nomes com hífen (CSS) viram camelCase (JS)
  • Exemplo: font-size vira fontSize
  • Exemplo: background-color vira backgroundColor

4. Eventos e Interatividade

O que são Eventos

Eventos são ações do usuário que o JavaScript pode escutar e reagir:

  • click: Clique do mouse
  • keydown/keyup: Pressionamento de tecla
  • change: Mudança de valor em input
  • submit: Submissão de formulário
  • mouseover/mouseout: Entrada/saída do mouse

addEventListener — A Forma Moderna

addEventListener é a forma mais robusta de monitorar eventos:

  • Permite múltiplos "ouvintes" no mesmo elemento
  • Um não sobrescreve o outro (diferente de onclick)
  • Mais controle (pode remover com removeEventListener)
  • Padrão profissional e seguro

5. Segurança: Prevenção de Injeção

O Risco XSS (Cross-Site Scripting)

Quando dados vêm do usuário ou de APIs externas, há risco de injeção maliciosa:

❌ RISCO:
elemento.innerHTML = dadoDoUsuario;
Se usuário digita: <img src=x onerror="alert('Hackeado!)">
O script malicioso é executado!

✓ SEGURO:
elemento.textContent = dadoDoUsuario;
Apenas texto é inserido, tags são escapadas

Proteção

  • Usar textContent em vez de innerHTML para dados do usuário
  • Validar e sanitizar entradas no servidor (back-end)
  • Usar bibliotecas como DOMPurify se innerHTML é necessário
  • Nunca confiar em dados do cliente — sempre validar no servidor

6. Fluxo de Execução Típico

  1. Usuário interage: Clica botão, digita texto, etc
  2. Evento é disparado: Browser cria evento (click, keydown, etc)
  3. Listener escuta: addEventListener detecta evento
  4. Callback executa: Função associada é chamada
  5. DOM é manipulado: JavaScript muda conteúdo/estilo
  6. Tela atualiza: Browser renderiza mudanças (reflow/repaint)

ANÁLISE TÉCNICA DO CÓDIGO

Estrutura Geral

O código foi desenvolvido seguindo padrões profissionais de desenvolvimento web, separando HTML, CSS e JavaScript de forma clara e mantível.

Arquitetura

  • HTML: Estrutura semântica com IDs descritivos
  • CSS: Design system com variáveis (temas)
  • JavaScript: Funções puras e reutilizáveis

Técnicas CSS Avançadas Utilizadas

1. Variáveis CSS (Custom Properties)

O código usa :root { --cor-fundo, --cor-texto, etc } para criar um sistema de temas:

Variáveis CSS
/* Definir variáveis no :root */
:root {
  --cor-fundo: #ffffff;
  --cor-texto: #333333;
}

/* Usar em qualquer elemento */
body {
  background-color: var(--cor-fundo);
  color: var(--cor-texto);
}

2. Tema Escuro com Seletor de Classe

Ao adicionar classe .dark-theme, as variáveis são sobrescritas:

Dark Theme Override
/* Modo escuro — sobrescreve variáveis */
:root.dark-theme {
  --cor-fundo: #1a1a2e;  /* Azul escuro */
  --cor-texto: #ffffff;  /* Branco */
}

3. Transições Suaves

CSS transitions criam animações automáticas quando propriedades mudam:

Transições
body {
  transition: background-color 0.3s ease,
              color 0.3s ease;
}

4. Toggle de Classes com Opacidade

Usar opacity + max-height para animação de mostrar/esconder:

Show/Hide Animation
/* Estado normal */
#secao-ocultavel {
  opacity: 1;
  max-height: 1000px;
  transition: opacity 0.3s ease;
}

/* Classe adicionada por JavaScript */
#secao-ocultavel.escondida {
  opacity: 0;
  max-height: 0;
}

Técnicas JavaScript Utilizadas

1. Variáveis Globais para Estado

As variáveis contador, temaEscuroAtivo e secaoVisivel mantêm estado entre chamadas de função:

Global State
// Mantém contador mesmo após função terminar
let contador = 0;
let temaEscuroAtivo = false;
let secaoVisivel = true;

2. Validação de Entrada

Todas as funções validam entrada antes de processar:

Validação
const nome = inputNome.value.trim();
if (nome === '') {
  alert('Por favor, digite seu nome!');
  return;
}

3. DOM Manipulation Patterns

Padrões recorrentes no código:

  • getElementById → .value: Ler entrada de user
  • getElementById → .textContent: Atualizar texto
  • .classList.toggle(): Alternar classes CSS
  • .classList.add/remove(): Adicionar/remover classes específicas

Exemplos de Funções Chave

1. Boas-Vindas Personalizadas

exibirBoasVindas()
function exibirBoasVindas() {
  // 1. Encontrar elemento e ler valor
  const nome = document.getElementById('nome-usuario').value.trim();

  // 2. Validar
  if (nome === '') return;

  // 3. Modificar conteúdo
  const msg = document.getElementById('mensagem-boas-vindas');
  msg.textContent = `🎉 Bem-vindo, ${nome}!`;

  // 4. Adicionar classe CSS (transition ativa automaticamente)
  msg.classList.add('visivel');
}

2. Contador com Validação Condicional

atualizarDisplay()
function atualizarDisplay() {
  const display = document.getElementById('numero-contador');
  display.textContent = contador;

  // Mudar cor se negativo
  if (contador < 0) {
    display.classList.add('negativo');
  } else {
    display.classList.remove('negativo');
  }
}

3. Alternar Tema com Toggle

alternarTema()
function alternarTema() {
  // Encontrar raiz (<html>)
  const html = document.documentElement;

  // Toggle classe .dark-theme
  html.classList.toggle('dark-theme');

  // Atualizar flag e botão
  temaEscuroAtivo = !temaEscuroAtivo;
  const btn = document.getElementById('btn-tema');
  btn.textContent = temaEscuroAtivo ? '☀️ Tema Claro' : '🌙 Tema Escuro';
}

4. Toggle de Visibilidade

alternarVisibilidade()
function alternarVisibilidade() {
  const secao = document.getElementById('secao-ocultavel');

  // Toggle adiciona OU remove classe automaticamente
  secao.classList.toggle('escondida');

  // CSS transition faz animação suave automaticamente
  secaoVisivel = !secaoVisivel;
}

Pontos Fortes do Código

  • ✓ Separação de responsabilidades: Cada função tem um propósito claro
  • ✓ Reutilização: atualizarDisplay() é chamada por ambas as funções de contador
  • ✓ CSS-driven: Animações e estilos ficam em CSS, JS apenas adiciona/remove classes
  • ✓ Validação: Entrada do usuário é sempre validada
  • ✓ Acessibilidade: IDs descritivos facilitam manutenção
  • ✓ Comentários educacionais: Cada função explica o que faz

Melhorias Futuras Possíveis

1. Persistência com localStorage

Salvar estado do contador no navegador para recuperar ao recarregar:

localStorage.setItem('contador', contador);

2. Eventos Mais Complexos

botao.addEventListener('click', incrementarContador);

3. Validação Aprimorada

Limitar comprimento, apenas números, mensagens de erro inline

Para animações mais suaves em navegadores menos potentes

CONCEITOS-CHAVE DO DOM

1. Document Object Model (DOM)

Conceito Explicação
document Objeto global que representa a página HTML inteira
Node Qualquer elemento na árvore (tags, atributos, textos)
Element Especificamente uma tag HTML (<div>, <button>, etc)
NodeList Coleção estática de nodes (resultado de querySelectorAll)
HTMLCollection Coleção dinâmica de elements (resultado de getElementsBy*)

2. Propriedades de Acesso e Manipulação

Propriedade O que faz Exemplo
.textContent Lê/altera apenas TEXTO visível elemento.textContent = "Novo texto"
.innerHTML Lê/altera HTML (permite tags) elemento.innerHTML = "<b>Bold</b>"
.value Lê/altera valor de inputs input.value = "José"
.classList Manipula classes CSS elemento.classList.add('ativa')
.style Altera estilos inline elemento.style.color = "red"
.id / .className Lê/altera ID ou class atributo elemento.id = "novo-id"

3. Métodos classList (mais usado)

Método O que faz Caso de Uso
.add('classe') Adiciona classe (não remove outras) Marcar como "ativa"
.remove('classe') Remove classe específica Desmarcar como "ativa"
.toggle('classe') Adiciona se não existe, remove se existe Botão de on/off
.contains('classe') Verifica se tem classe (retorna boolean) Condicional baseado em classe

4. Operadores Booleanos Essenciais

Boolean Logic
// AND (&&) — AMBAS precisam ser TRUE
if (nome !== '' && numero > 0) { ... }

// OR (||) — QUALQUER UMA pode ser TRUE
if (temaEscuro || modoNoturno) { ... }

// NOT (!) — Inverte resultado
if (!secaoVisivel) { ... }  // Se NÃO visível

// Negação dupla (!!) — Converte para boolean
const ativo = !!valor;  // força true ou false

5. Template Literals (Interpolação de Strings)

Template Literals
// ✗ Concatenação tradicional (verbosa)
"Bem-vindo, " + nome + "!"

// ✓ Template literal (legível)
`Bem-vindo, ${nome}!`

// Múltiplas linhas
`Contador: ${contador}
Tema: ${temaEscuroAtivo ? 'Escuro' : 'Claro'}`

Checklist de Implementação

  • ☐ Script está no final do <body>

  • ☐ Funções validam entrada do usuário

  • ☐ CSS define animações (não JavaScript)
  • ☐ Variáveis globais rastreiam estado
  • ☐ Comentários explicam lógica complexa
