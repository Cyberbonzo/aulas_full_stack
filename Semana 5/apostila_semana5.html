📖 Resumo da Aula

A quinta aula da mentoria foi dedicada inteiramente ao aprendizado de Banco de Dados e SQL, abordando desde conceitos teóricos fundamentais até a prática de comandos em um ambiente simulado.

Os principais tópicos técnicos e práticos discutidos foram:

  1. Fundamentos de Banco de Dados
    • Dados vs. Informação: Diferenciação entre dado (elemento básico sem contexto), informação (dados associados que geram significado) e conhecimento (discernimento aplicado sobre a informação).
    • SGBD: Sistema Gerenciador de Banco de Dados é o software que gerencia o acesso e organização dos dados (exemplos: DB2, PostgreSQL, MySQL, SQLite).
    • Papel da API: Atua como ponte ou "tradutor" que permite comunicação segura entre aplicação (como PHP) e banco de dados, abstraindo a complexidade do motor.
  2. Linguagem SQL e seus Subconjuntos
    • DQL (Consulta): Comando SELECT para extrair informações.
    • DML (Manipulação): Comandos INSERT, UPDATE e DELETE para gerenciar registros.
    • DDL (Definição): Comandos CREATE (criar tabelas), ALTER (modificar estrutura) e DROP (excluir objetos).
    • DCL (Controle): Gerenciamento de permissões de usuários.
    • DTL (Transação): Comandos BEGIN, COMMIT e ROLLBACK para garantir execução integra ou cancelamento em caso de erro.
  3. Prática de CRUD e Estrutura
    • Criação e Tipagem: Comando CREATE TABLE com tipos como INTEGER e VARCHAR.
    • Chaves (Keys): Chave Primária (identificador único) e Chave Estrangeira (vinculação entre tabelas).
    • Filtros e Consultas: Cláusula WHERE para filtros e asterisco (*) para selecionar todas as colunas.
    • Junções (Joins): INNER JOIN (dados presentes em ambas tabelas) e LEFT JOIN (mantém dados da tabela esquerda).
  4. Segurança e Consistência
    • Cuidado com WHERE: Risco extremo de UPDATE/DELETE sem WHERE apagaria todos os registros.
    • Transações: Blocos de transação para testar com ROLLBACK antes de COMMIT.
💡 Conclusão: A mentoria enfatizou a importância da documentação como melhor amiga do desenvolvedor e a necessidade de praticar esses comandos para fixar a lógica, que será usada futuramente integrada ao PHP.

🗂️ Exercício 1: CRUD Básico (Clientes e Pedidos)

Este exercício prático trabalha com duas tabelas relacionadas: CLIENTES e PEDIDOS. Demonstra os conceitos fundamentais de Create, Read, Update e Delete.

1️⃣ Criação de Tabelas (DDL)

CRIAR TABELA CLIENTES
-- Cria tabela para armazenar informações de clientes CREATE TABLE clientes ( -- ID única para cada cliente (chave primária) id INT PRIMARY KEY AUTO_INCREMENT, -- Nome do cliente (obrigatório, até 100 caracteres) nome VARCHAR(100) NOT NULL, -- Email único do cliente (não pode haver duplicação) email VARCHAR(100) UNIQUE, -- Telefone de contato (até 15 caracteres) telefone VARCHAR(15), -- Data de registro (preenchida automaticamente com data/hora atual) data_cadastro TIMESTAMP DEFAULT CURRENT_TIMESTAMP );
📌 Tipos de Dados Utilizados:
  • INT: Número inteiro para IDs e identificadores
  • VARCHAR(n): Texto de tamanho variável (até n caracteres)
  • TIMESTAMP: Data e hora, com valor padrão automático
CRIAR TABELA PEDIDOS
-- Cria tabela para armazenar pedidos dos clientes CREATE TABLE pedidos ( -- ID única para cada pedido (chave primária) id INT PRIMARY KEY AUTO_INCREMENT, -- ID do cliente que fez o pedido (chave estrangeira vinculada à tabela clientes) cliente_id INT NOT NULL, -- Descrição do produto pedido descricao VARCHAR(255), -- Valor do pedido (10 dígitos, sendo 2 casas decimais) valor DECIMAL(10, 2), -- Status padrão "Pendente" até ser processado status VARCHAR(50) DEFAULT 'Pendente', -- Data e hora do pedido (automática) data_pedido TIMESTAMP DEFAULT CURRENT_TIMESTAMP, -- CHAVE ESTRANGEIRA: vincula cada pedido a um cliente específico FOREIGN KEY (cliente_id) REFERENCES clientes(id) );
⚠️ Importante - Chaves Estrangeiras: A cláusula FOREIGN KEY garante que cada pedido está vinculado a um cliente válido. Sem isso, você poderia criar pedidos órfãos.

2️⃣ Inserção de Dados (DML - INSERT)

INSERIR CLIENTES
-- Insere múltiplos clientes em uma única operação INSERT INTO clientes (nome, email, telefone) VALUES ('Cliente 1', 'cliente1@email.com', '11987654321'), ('Cliente 2', 'cliente2@email.com', '11912345678'), ('Cliente 3', 'cliente3@email.com', '11998765432'), ('Cliente 4', 'cliente4@email.com', '11987654310'); -- Note: ID será preenchido automaticamente (1, 2, 3, 4) -- Note: data_cadastro será preenchida com a data/hora atual
INSERIR PEDIDOS
-- Insere pedidos vinculando-os aos clientes por cliente_id INSERT INTO pedidos (cliente_id, descricao, valor) VALUES (1, 'Notebook Dell', 3500.00), -- Pedido do Cliente 1 (1, 'Mouse Logitech', 150.00), -- Outro pedido do Cliente 1 (2, 'Teclado Mecânico', 450.00), -- Pedido do Cliente 2 (3, 'Monitor LG 27"', 1200.00), -- Pedido do Cliente 3 (2, 'Webcam HD', 250.00); -- Outro pedido do Cliente 2

3️⃣ Consultas Simples (DQL - SELECT)

SELECIONAR DADOS
-- Mostra TODOS os clientes (asterisco = todas as colunas) SELECT * FROM clientes; -- Mostra apenas nome e email dos clientes (seleção de colunas específicas) SELECT nome, email FROM clientes; -- Filtra pedidos com valor maior que R$ 500 usando WHERE SELECT * FROM pedidos WHERE valor > 500; -- Conta quantos clientes existem (função agregada COUNT) SELECT COUNT(*) AS total_clientes FROM clientes; -- Conta quantos pedidos cada cliente fez (agrupamento com GROUP BY) SELECT cliente_id, COUNT(*) AS total_pedidos FROM pedidos GROUP BY cliente_id;

4️⃣ Junções (JOINS) - Relacionando Tabelas

INNER JOIN - Clientes com Pedidos
-- INNER JOIN: mostra APENAS clientes que têm pedidos SELECT c.nome, c.email, p.descricao, p.valor FROM clientes c INNER JOIN pedidos p ON c.id = p.cliente_id; -- Abreviações: 'c' para clientes, 'p' para pedidos (aliases)
✅ INNER JOIN: Retorna apenas registros que têm correspondência em AMBAS as tabelas. Se um cliente não tiver pedidos, ele não aparecerá.
LEFT JOIN - Todos os Clientes
-- LEFT JOIN: mostra TODOS os clientes, mesmo sem pedidos SELECT c.nome, c.email, COUNT(p.id) AS total_pedidos, SUM(p.valor) AS valor_total FROM clientes c LEFT JOIN pedidos p ON c.id = p.cliente_id GROUP BY c.id, c.nome, c.email; -- COUNT(p.id): conta pedidos (NULL não é contado) -- SUM(p.valor): soma valores dos pedidos
✅ LEFT JOIN: Retorna TODOS os registros da tabela da esquerda (clientes), mesmo que não existam correspondências na tabela direita (pedidos).

5️⃣ Atualização (DML - UPDATE)

ATUALIZAR REGISTROS
-- Atualiza email de UM cliente específico (WHERE é CRÍTICO!) UPDATE clientes SET email = 'cliente1.novo@email.com' WHERE id = 1; -- Atualiza status de um pedido específico UPDATE pedidos SET status = 'Entregue' WHERE id = 1; -- Atualiza MÚLTIPLOS pedidos de um cliente UPDATE pedidos SET status = 'Processando' WHERE cliente_id = 2; -- Cálculo: aumenta valor de todos os pedidos em 10% UPDATE pedidos SET valor = valor * 1.10;
⚠️ CUIDADO CRÍTICO: Note a última instrução SEM WHERE! Isso afeta TODOS os pedidos. Sempre teste com SELECT antes de UPDATE!

6️⃣ Deleção (DML - DELETE)

DELETAR REGISTROS
-- Deleta UM pedido específico DELETE FROM pedidos WHERE id = 5; -- Deleta TODOS os pedidos de um cliente específico DELETE FROM pedidos WHERE cliente_id = 3; -- Deleta um cliente E seus pedidos (ordem importante!) -- Primeiro deleta os pedidos (chave estrangeira) DELETE FROM pedidos WHERE cliente_id = 4; -- Depois deleta o cliente DELETE FROM clientes WHERE id = 4;
⚠️ NUNCA ESQUEÇA O WHERE! Um DELETE sem WHERE deleta TODOS os registros da tabela!

7️⃣ Transações (DTL - Segurança)

TRANSAÇÕES COM BEGIN/COMMIT/ROLLBACK
-- Inicia uma transação (bloco de operações) BEGIN; -- Insere cliente de teste INSERT INTO clientes (nome, email, telefone) VALUES ('Teste Transação', 'teste@email.com', '11999999999'); -- Insere pedido para este cliente INSERT INTO pedidos (cliente_id, descricao, valor) VALUES (1, 'Teste Produto', 100.00); -- Se tudo está OK, CONFIRMA as operações: COMMIT; -- OU se algo deu errado, DESFAZ tudo: -- ROLLBACK;
✅ Transações: Permitem testar operações em memória. Se algo der errado, ROLLBACK desfaz tudo. Se tudo OK, COMMIT grava permanentemente.

🎯 Exercício 2: Desafio SQL (Produtos e Estoque)

Este exercício mais avançado trabalha com gestão de produtos e movimentações de estoque, introduzindo conceitos como CASE, agrupamento avançado e cálculos complexos.

1️⃣ Estrutura de Produtos e Estoque

CRIAR TABELAS DE PRODUTOS E MOVIMENTAÇÃO
-- Tabela para armazenar informações de produtos CREATE TABLE produtos ( id INT PRIMARY KEY AUTO_INCREMENT, nome VARCHAR(100) NOT NULL, categoria VARCHAR(50), preco DECIMAL(10, 2) NOT NULL, estoque INT DEFAULT 0, data_criacao TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- Tabela para rastrear HISTÓRICO de movimentações (entrada/saída) CREATE TABLE estoque_movimentacao ( id INT PRIMARY KEY AUTO_INCREMENT, produto_id INT NOT NULL, tipo VARCHAR(20), -- 'Entrada' ou 'Saída' quantidade INT, data_movimentacao TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (produto_id) REFERENCES produtos(id) );

2️⃣ Consultas com Lógica Condicional (CASE)

CASE - Classificação Dinâmica de Preços
-- Classifica produtos por faixa de preço usando CASE SELECT nome, preco, CASE WHEN preco < 300 THEN 'Econômico' WHEN preco BETWEEN 300 AND 700 THEN 'Intermediário' ELSE 'Premium' END AS faixa_preco FROM produtos ORDER BY preco; -- CASE funciona como um IF/ELSE: avalia cada WHEN até encontrar verdadeiro
CASE - Nível de Estoque com Alertas
-- Verifica saúde do estoque com alertas visuais SELECT nome, estoque, CASE WHEN estoque < 5 THEN 'Crítico - REABASTECER!' WHEN estoque < 10 THEN 'Baixo' ELSE 'Normal' END AS nivel_estoque FROM produtos; -- Útil para dashboards de monitoramento de inventário

3️⃣ Análise de Vendas com Agregação

ANÁLISE DE VENDAS POR PRODUTO
-- Conta quantas vendas (saídas) cada produto teve SELECT p.nome, p.categoria, COUNT(em.id) AS total_vendas, SUM(em.quantidade) AS total_quantidade_vendida FROM produtos p LEFT JOIN estoque_movimentacao em ON p.id = em.produto_id AND em.tipo = 'Saída' GROUP BY p.id, p.nome, p.categoria ORDER BY total_quantidade_vendida DESC; -- LEFT JOIN garante que produtos sem vendas também apareçam -- AND em.tipo = 'Saída': filtra apenas saídas (vendas), não entradas
VALOR TOTAL EM ESTOQUE
-- Calcula o valor total de inventário (quantidade * preço unitário) SELECT nome, preco, estoque, (preco * estoque) AS valor_total_estoque FROM produtos ORDER BY valor_total_estoque DESC; -- Importante para fins contábeis e análise financeira

4️⃣ Atualização Baseada em Condições

UPDATE COM LÓGICA CONDICIONAL
-- Aplica desconto de 10% apenas em produtos de Audio UPDATE produtos SET preco = preco * 0.90 WHERE categoria = 'Audio'; -- Reduz estoque de produtos Premium (preço > R$ 600) UPDATE produtos SET estoque = estoque - 1 WHERE preco > 600 AND estoque > 0; -- WHERE com AND: múltiplas condições (ambas devem ser verdadeiras)

5️⃣ Deleção Segura com Transação

DELETAR COM SEGURANÇA
-- Inicia transação para testar exclusão antes de confirmar BEGIN; -- Remove movimentações do produto DELETE FROM estoque_movimentacao WHERE produto_id = 5; -- Remove o produto DELETE FROM produtos WHERE id = 5; -- Verifica resultado SELECT * FROM produtos; SELECT * FROM estoque_movimentacao; -- Se tudo OK: COMMIT; (descomente) -- Se erro: ROLLBACK; (descomente)

6️⃣ Criação de View (Visão/Relatório)

CREATE VIEW - Relatório de Estoque
-- Cria uma "tabela virtual" para simplificar consultas frequentes CREATE VIEW relatorio_estoque AS SELECT nome, categoria, preco, estoque, (preco * estoque) AS valor_total FROM produtos WHERE estoque > 0 ORDER BY estoque ASC; -- Usar a view (consultar como se fosse uma tabela normal) SELECT * FROM relatorio_estoque; -- Vantagem: reutilizar queries complexas sem reescrever
✅ VIEW: É como salvar uma consulta complexa com nome. Você a consulta como tabela normal, mas os dados sempre são atualizados em tempo real.

7️⃣ Transação com Múltiplas Operações

TRANSAÇÃO - Simular Venda
-- Simula processo completo de venda BEGIN; -- Reduz estoque do produto UPDATE produtos SET estoque = estoque - 1 WHERE id = 1; -- Registra a movimentação (saída) INSERT INTO estoque_movimentacao (produto_id, tipo, quantidade) VALUES (1, 'Saída', 1); -- Se tudo correr bem: COMMIT; (descomente) -- Se der erro: ROLLBACK; (descomente)

📋 Gabarito - Perguntas Práticas

RESPOSTAS ESPERADAS
-- 1. Quantos produtos existem no banco? SELECT COUNT(*) AS total_produtos FROM produtos; -- 2. Qual é o produto mais caro? SELECT nome, preco FROM produtos ORDER BY preco DESC LIMIT 1; -- 3. Qual categoria tem mais produtos? SELECT categoria, COUNT(*) AS total FROM produtos GROUP BY categoria ORDER BY total DESC; -- 4. Qual é o estoque total de toda a loja? SELECT SUM(estoque) AS estoque_total FROM produtos; -- 5. Qual produto teve mais movimentações? SELECT p.nome, COUNT(em.id) AS total_movimentacoes FROM produtos p LEFT JOIN estoque_movimentacao em ON p.id = em.produto_id GROUP BY p.id, p.nome ORDER BY total_movimentacoes DESC;

🎓 Resumo de Conceitos-Chave

  1. DDL (Data Definition Language): CREATE, ALTER, DROP
    • Define a ESTRUTURA das tabelas
    • CREATE TABLE - criar tabelas
    • ALTER TABLE - modificar estrutura
    • DROP - excluir objetos
  2. DML (Data Manipulation Language): INSERT, UPDATE, DELETE
    • Manipula os DADOS dentro das tabelas
    • INSERT - adicionar registros
    • UPDATE - modificar registros
    • DELETE - remover registros
  3. DQL (Data Query Language): SELECT
    • Consulta e extrai dados
    • SELECT * - todas as colunas
    • WHERE - filtros
    • JOIN - relacionar tabelas
    • GROUP BY - agrupamentos
    • ORDER BY - ordenação
  4. DTL (Data Transaction Language): BEGIN, COMMIT, ROLLBACK
    • Garante consistência de operações
    • BEGIN - inicia transação
    • COMMIT - confirma operações
    • ROLLBACK - desfaz operações
  5. Chaves:
    • PRIMARY KEY: Identificador único de cada registro
    • FOREIGN KEY: Vincula tabelas relacionadas
  6. JOINS:
    • INNER JOIN: Apenas registros com correspondência
    • LEFT JOIN: Todos da esquerda + correspondências da direita
    • RIGHT JOIN: Todos da direita + correspondências da esquerda
🔴 Regras Críticas de Segurança:
  • SEMPRE use WHERE ao atualizar ou deletar (teste com SELECT primeiro)
  • Respeite chaves estrangeiras ao deletar (delete dependentes primeiro)
  • Use transações para operações críticas
  • Faça backups regularmente
