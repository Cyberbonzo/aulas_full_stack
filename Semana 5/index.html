📋 1. Atividades Práticas

Atividade 1: Repetição de CRUD Básico

Arquivo: exercicio_crud_basico.sql

Exercícios práticos focando em:

  • CREATE TABLE: Definir colunas, tipos de dados, chaves primárias e estrangeiras
  • INSERT: Popular tabelas com dados de clientes e pedidos
  • SELECT e JOIN: Consultas simples, INNER JOIN e LEFT JOIN
  • UPDATE: Alterar emails, status e outros campos
  • DELETE: Remover registros com cuidado ao usar WHERE
  • ALTER TABLE: Adicionar novas colunas (ex: status)
  • Transações: BEGIN, COMMIT, ROLLBACK para garantir consistência
Tabelas principais: clientes e pedidos com relacionamento 1:N

Atividade 2: Desafio SQL (Conceitual)

Arquivo: atividade_2_desafio_sql.sql

Exercícios com desafios diferentes para reforçar aprendizado:

  • Desafio 1: Criar sistema de PRODUTOS e ESTOQUE com histórico
  • Desafio 2: Usar CASE para lógica condicional
  • Desafio 3: Análise de vendas com agregação (COUNT, SUM)
  • Desafio 4: UPDATE com condições
  • Desafio 5: DELETE com transações
  • Desafio 6: Junções complexas (3+ tabelas)
  • Desafio 7: Window Functions (recursos avançados)
  • Desafio 8: Criar e usar VIEWs
  • Desafio 9: Transações com múltiplas operações

🛠️ 2. Ferramentas para Praticar

Para Gerenciar Banco Local

  • HeidiSQL - Interface gráfica para MySQL/MariaDB (Windows)
  • RAID - Gerenciador de banco de dados alternativo

Para Testes Rápidos Online

  • SQL Lite Online - Editor SQL no navegador (sem instalar nada)
  • Replit SQL - Ambiente de programação com suporte a SQL
  • SQLFiddle - Teste queries em tempo real

💡 Dica: Use SQL Lite Online para começar rapidinho sem precisar instalar nada!

🎯 4. Como Usar os Arquivos SQL

Passo a Passo:

  1. Abra uma ferramenta SQL (HeidiSQL, SQL Lite Online, etc)
  2. Copie o código de exercicio_crud_basico.sql ou atividade_2_desafio_sql.sql
  3. Cole na ferramenta e execute os comandos em ordem
  4. Observe os resultados de cada query
  5. Experimente modificar os comandos para entender como funcionam
  6. Pratique transações: use BEGIN/COMMIT/ROLLBACK para testar sem medo

💡 Dicas de Estudo:

  • ✅ Sempre teste um SELECT antes de fazer UPDATE/DELETE
  • ✅ Use transações (BEGIN...COMMIT) para praticar sem "estragar" os dados
  • ✅ Pratique as JOINs várias vezes — é essencial!
  • ✅ Modifique os dados e veja o resultado mudar
  • ✅ Crie suas próprias queries além dos exercícios
