Algoritmo de Verificação de Notas

Semana 2 — Controle de Fluxo e Estrutura de Dados

1. Linguagem Natural

  1. Início do algoritmo.
  2. Entrada: Solicitar ao usuário que informe a nota do aluno (um número).
  3. Validação: Verificar se a nota informada está dentro do intervalo permitido, ou seja, se ela é maior ou igual a 0 e menor ou igual a 10.
    • Se a nota estiver fora desse intervalo, exibir a mensagem "Nota inválida. Por favor, informe um valor entre 0 e 10." e retornar ao passo 2.
  4. Classificação: Com a nota válida em mãos, verificar em qual faixa ela se enquadra, na seguinte ordem:
    • Se a nota for maior ou igual a 7, o aluno está Aprovado.
    • Caso contrário, se a nota for maior ou igual a 5 (e menor que 7), o aluno está Em Recuperação.
    • Caso contrário (nota menor que 5), o aluno está Reprovado.
  5. Saída: Exibir o resultado da classificação do aluno.
  6. Fim do algoritmo.

2. Pseudocódigo

ALGORITMO VerificacaoDeNota

  // Declaração de variável
  VARIÁVEL nota: REAL
  /*
     Em JavaScript : let nota;  ou  const nota = 0;
     Em PHP        : $nota;     (variáveis iniciam com $)
     Em Python     : nota = 0   (sem declaração explícita de tipo)
     Em Java/C#    : double nota;
  */

  // Entrada e validação com laço de repetição
  REPITA
    ESCREVA("Informe a nota do aluno (0 a 10): ")
    LEIA(nota)
    /*
       JavaScript: nota = parseFloat(prompt("Informe a nota:"));
       PHP       : $nota = (float) $_POST['nota'];
    */

    SE (nota < 0 OU nota > 10) ENTÃO
    /*
       Operadores de comparação:
         <   menor que          |  >   maior que
         <=  menor ou igual     |  >=  maior ou igual
         ==  igual (valor)      |  !=  diferente
         === igual (valor+tipo) — exclusivo do JS/PHP

       Operadores lógicos:
         OU  → OR  → ||  (JavaScript/PHP/C)
         E   → AND → &&  (JavaScript/PHP/C)
         NÃO → NOT → !   (JavaScript/PHP/C)

       NOTA: No HTML dentro de 
, os sinais < e > devem
       ser escritos como entidades HTML (< e >) para não
       serem confundidos com tags HTML pelo navegador.
    */
      ESCREVA("Nota inválida. Informe um valor entre 0 e 10.")
    FIM SE

  ATÉ QUE (nota >= 0 E nota <= 10)
  /*
     Este é um laço de pós-condição: o bloco REPITA executa
     pelo menos UMA VEZ antes de verificar a condição.

     Equivalente em JavaScript:
       do {
         nota = parseFloat(prompt("Informe a nota:"));
         if (nota < 0 || nota > 10) {
           alert("Nota inválida.");
         }
       } while (nota < 0 || nota > 10);

     Equivalente em PHP:
       do {
         $nota = (float) $_POST['nota']; // em formulário real
         if ($nota < 0 || $nota > 10) {
           echo "Nota inválida.";
         }
       } while ($nota < 0 || $nota > 10);
  */

  // Classificação por estrutura condicional encadeada
  SE (nota >= 7) ENTÃO
    ESCREVA("Resultado: Aprovado")

  SENÃO SE (nota >= 5) ENTÃO
    ESCREVA("Resultado: Em Recuperação")
    // Não precisamos checar nota < 7 aqui porque,
    // se chegou neste ponto, o SE anterior já foi falso,
    // ou seja, nota < 7 já é garantido pelo fluxo.

  SENÃO
    ESCREVA("Resultado: Reprovado")
    // Todos os casos não cobertos acima (nota < 5)

  FIM SE
  /*
     Equivalente em JavaScript:
       if (nota >= 7) {
         console.log("Aprovado");
       } else if (nota >= 5) {
         console.log("Em Recuperação");
       } else {
         console.log("Reprovado");
       }

     Equivalente em PHP:
       if ($nota >= 7) {
         echo "Aprovado";
       } elseif ($nota >= 5) {  // PHP usa elseif (sem espaço)
         echo "Em Recuperação";
       } else {
         echo "Reprovado";
       }
  */

FIM ALGORITMO
