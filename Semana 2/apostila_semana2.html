A segunda aula da mentoria focou nos fundamentos da Lógica de Programação, base para o desenvolvimento de qualquer sistema. O objetivo central é aprender a transcrever a resolução de problemas para uma linguagem que o computador compreenda.

Seção 02

Algoritmos & Representações

As quatro formas de descrever a solução de um problema.

Um algoritmo é uma sequência finita, ordenada e não ambígua de passos que, quando seguida, resolve um problema. Todo algoritmo segue o ciclo:

Entrada → dados fornecidos pelo usuário ou sistema
Processamento → transformação dos dados por meio de regras
Saída → resultado apresentado ao usuário

Formas de Representação

1. Linguagem Natural

Descrição em português (ou qualquer idioma) sem regras formais de sintaxe. Vantagem: fácil de ler. Desvantagem: ambígua — frases podem ser interpretadas de formas diferentes.

Exemplo: "Se a nota for maior ou igual a 7, o aluno está aprovado."

2. Fluxograma

Representação visual com símbolos geométricos padronizados. Facilita visualizar o caminho percorrido pelo programa. Não vimos exemplos visuais nesta apostila, mas os símbolos básicos são:

  • Oval → Início / Fim
  • Retângulo → Processamento (instrução)
  • Losango → Decisão (condição)
  • Paralelogramo → Entrada / Saída de dados

3. Pseudocódigo

Linguagem intermediária: usa estruturas formais (variáveis, SE, REPITA, ESCREVA) mas sem a sintaxe rígida de uma linguagem real. É o mais próximo do código sem ser código. Esta apostila utiliza pseudocódigo para descrever o algoritmo da tarefa.

4. Linguagem de Programação

Código executável com sintaxe precisa. Um erro de sintaxe impede a execução. Exemplos: JavaScript, PHP, Python, Java, C#. Cada uma tem suas próprias regras, mas a lógica por trás é a mesma — por isso aprender lógica primeiro é fundamental.

💡 Analogia

Pense na lógica como a receita de bolo e na linguagem de programação como o fogão. A receita não muda — você pode preparar o mesmo bolo em qualquer fogão. Da mesma forma, a lógica do algoritmo é a mesma em qualquer linguagem.

Seção 03

Variáveis, Constantes e Tipos de Dados

Como os programas armazenam e gerenciam informações na memória.

Variáveis e Constantes

São espaços reservados na memória RAM do computador, identificados por um nome (identificador), que guardam valores durante a execução do programa.

Variável

O valor pode ser alterado durante a execução. Exemplo: a nota do aluno — recebemos do usuário e pode mudar se ele digitar errado.

Constante

O valor é fixo após a definição. Exemplos: PI = 3.14159, NOTA_APROVACAO = 7, NOTA_MAX = 10.

Tipagem: Estática vs. Dinâmica

A tipagem define quando e como o tipo de dado de uma variável é verificado:

  • Estática (Java, C#): o tipo é declarado explicitamente no código e verificado em tempo de compilação. Ex: double nota = 8.5;
  • Dinâmica (PHP, JavaScript, Python): o tipo é inferido pelo valor atribuído, em tempo de execução. Ex: $nota = 8.5; — PHP entende que é um float.

Tipos de Dados Primitivos

Tipo Descrição Exemplos de valores Em JS / PHP
int Número inteiro (sem decimais) 0, 5, -3, 100 let x = 5 / $x = 5
float / real Número com casas decimais 3.14, 8.5, -0.75 let n = 8.5 / $n = 8.5
string Sequência de caracteres (texto) "Aprovado", "Usuário" "texto" / 'texto'
char Um único caractere 'A', 'z', '9' Em JS/PHP, é uma string de tamanho 1
boolean Verdadeiro ou falso true, false true / false

📌 Por que tipos importam?

O tipo determina quanta memória é usada e quais operações são permitidas. "5" + 3 em JavaScript resulta em "53" (concatenação de string), não em 8. Entender tipos evita esse tipo de bug difícil de depurar. Use parseFloat() ou parseInt() para converter strings em números.

Em JavaScript, == faz coerção de tipo (converte um dos lados antes de comparar). "7" == 7 é true. Isso pode causar bugs silenciosos. Use sempre === para comparações seguras em JavaScript e PHP.

Seção 05

Controle de Fluxo

Estruturas que permitem desvios condicionais e repetições no programa.

Decisão: if / else if / else

Permite executar blocos de código diferentes dependendo de uma condição.

decisao.js 
// Estrutura básica
if (condicao) {
  // executado se condicao for VERDADEIRA
} else if (outraCondicao) {
  // executado se a primeira for falsa E esta for verdadeira
} else {
  // executado se TODAS as condições anteriores forem falsas
}

// Aplicado à tarefa:
if (nota >= 7) {
  console.log("Aprovado");
} else if (nota >= 5) {
  console.log("Em Recuperação");
} else {
  console.log("Reprovado");
}

Repetição: for, while e do...while

LoopQuando usarExecuta ao menos uma vez?
for Quando se sabe quantas vezes repetir (contador fixo) Não (se o contador já não satisfaz a condição)
while Quando a condição é verificada antes de cada iteração Não (se condição inicial for falsa, nunca executa)
do...while Quando o bloco deve rodar ao menos uma vez antes da verificação Sim — sempre executa ao menos uma vez
loops.js — do...while (usado na tarefa) 
// do...while: ideal para validação de entrada de dados.
// O usuário DEVE fornecer uma entrada ao menos uma vez.
let nota;

do {
  nota = parseFloat(prompt("Informe a nota (0 a 10):"));

  if (nota < 0 || nota > 10 || isNaN(nota)) {
    alert("Valor inválido. Tente novamente.");
  }
} while (nota < 0 || nota > 10 || isNaN(nota));

// Após o loop, temos certeza que nota está entre 0 e 10.

💡 Por que do...while para validação?

Com while, precisaríamos inicializar nota com um valor inválido artificial (como -1) para forçar a primeira execução. Com do...while, a leitura do usuário acontece naturalmente antes da verificação — código mais limpo e intuitivo.

Seção 06

A Tarefa Explicada

Decomposição do problema em regras de negócio e solução em linguagem natural.

O Problema

Criar um algoritmo que receba a nota de um aluno, valide se está no intervalo permitido e classifique o aluno em uma das três situações.

Regras de Negócio

Condição Faixa de Nota Situação
nota < 0 ou nota > 10 Fora do intervalo Nota inválida — solicitar novamente
nota >= 7 7,0 — 10,0 Aprovado
nota >= 5 (e < 7) 5,0 — 6,9 Em Recuperação
nota < 5 0,0 — 4,9 Reprovado

Solução em Linguagem Natural (passo a passo)

  1. Início do algoritmo.
  2. Entrada: Solicitar ao usuário que informe a nota do aluno.
  3. Validação: Verificar se a nota está entre 0 e 10 (inclusive).
    • Se estiver fora do intervalo: exibir mensagem de erro e voltar ao passo 2.
    • Se estiver dentro: continuar para o passo 4.
  4. Classificação: Verificar a faixa da nota na seguinte ordem:
    • Se nota ≥ 7 → Aprovado
    • Caso contrário, se nota ≥ 5 → Em Recuperação
    • Caso contrário → Reprovado
  5. Saída: Exibir o resultado da classificação.
  6. Fim do algoritmo.

📌 Por que a ordem das condições importa?

Na classificação, verificamos nota >= 7 primeiro. Se chegamos ao else if (nota >= 5), já sabemos — pelo fluxo — que nota < 7. Não precisamos escrever nota >= 5 && nota < 7. Isso é uso eficiente do controle de fluxo: condições anteriores já descartam casos.

Seção 07

Pseudocódigo Comentado

A lógica da tarefa linha a linha, com explicações de cada estrutura.

verificacao-notas.pseudo 
ALGORITMO VerificacaoDeNota
  // Nomear o algoritmo é boa prática de documentação.

  VARIÁVEL nota: REAL
    // Declaramos a variável antes de usá-la.
  // Tipo REAL = aceita números decimais (ex: 6.5, 7.0).
  // Em linguagens dinâmicas (JS/PHP) esta linha não é
  // necessária — a variável é criada ao receber o valor.

  REPITA
      // Início do laço de pós-condição (do...while).
    // O bloco dentro de REPITA executa ao menos UMA vez
    // antes de a condição ser verificada no ATÉ QUE.

    ESCREVA("Informe a nota do aluno (0 a 10): ")
    LEIA(nota)
        // ESCREVA = console.log / echo / print
    // LEIA    = prompt() / $_POST / input()

    SE (nota < 0 OU nota > 10) ENTÃO
      ESCREVA("Nota inválida. Informe um valor entre 0 e 10.")
    FIM SE
        // Se a nota é inválida, avisamos e o laço reinicia.

  ATÉ QUE (nota >= 0 E nota <= 10)
    // O laço PARA quando nota estiver no intervalo [0, 10].
  // Enquanto a nota for inválida, o laço continua.

  SE (nota >= 7) ENTÃO
    ESCREVA("Resultado: Aprovado")

  SENÃO SE (nota >= 5) ENTÃO
    ESCREVA("Resultado: Em Recuperação")
        // Aqui já sabemos que nota < 7 (fluxo garante).
    // Portanto, nota >= 5 implica 5 <= nota < 7.

  SENÃO
    ESCREVA("Resultado: Reprovado")
        // Todos os casos restantes: nota < 5.

  FIM SE

FIM ALGORITMO

Classificação em JavaScript

classificacao.js 
let nota;
do {
  nota = parseFloat(prompt("Nota (0-10):"));
} while (isNaN(nota) || nota < 0 || nota > 10);

if      (nota >= 7) alert("Aprovado");
else if (nota >= 5) alert("Em Recuperação");
else               alert("Reprovado");

Classificação em PHP

classificacao.php 
<?php
// PHP processa no servidor. Recebe a nota via formulário HTML.
if ($_SERVER["REQUEST_METHOD"] === "POST") {
  $nota = filter_input(INPUT_POST, 'nota', FILTER_VALIDATE_FLOAT);

  if ($nota === false || $nota < 0 || $nota > 10) {
    echo "Nota inválida.";
  } elseif ($nota >= 7) {
    echo "Aprovado";
  } elseif ($nota >= 5) {
    echo "Em Recuperação";
  } else {
    echo "Reprovado";
  }
}
?>

💡 PHP vs JavaScript: onde cada um roda?

PHP roda no servidor — o usuário nunca vê o código PHP, apenas o HTML resultante. Acessa banco de dados, gerencia sessões e arquivos.
JavaScript roda no navegador — manipula a página em tempo real sem recarregar. Dominar os dois é o que define um desenvolvedor Full Stack.

O HTML da Entrega

Estrutura da página estática entregue na tarefa e explicação de cada tag utilizada.

Por que HTML estático?

Nesta tarefa não há lógica funcional em JavaScript. O objetivo é apenas apresentar o algoritmo em formato legível dentro de um navegador, usando HTML como ferramenta de formatação de texto — assim como se usaria Word ou Google Docs.

📌 Semântica vs. Apresentação

Use <strong> para importância semântica e font-weight: bold para negrito puramente visual. Use <em> para ênfase semântica e font-style: italic para itálico visual. Leitores de tela interpretam a semântica — não o visual.

inline > ID > classe > elemento
!important força prioridade máxima (evite).

Box Model — o modelo de caixa

Todo elemento HTML é uma caixa retangular com quatro camadas, de dentro para fora:

content → o conteúdo (texto, imagem)
padding → espaço interno entre o conteúdo e a borda
border → a borda do elemento
margin → espaço externo entre este elemento e os vizinhos

Com box-sizing: border-box, padding e borda são incluídos na largura/altura declarada — muito mais intuitivo para layouts.

💡 Dica de impressão / PDF

Use @media print { } para aplicar estilos somente na impressão. Remova sombras, defina largura em mm, use page-break-after: always para forçar quebra de página entre seções. É exatamente assim que esta apostila foi criada.
