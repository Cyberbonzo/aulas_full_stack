1. O que é ser Full Stack e a Visão de Negócio

Definição: Ser um desenvolvedor Full Stack significa conhecer todas as camadas da aplicação — telas, APIs, banco de dados e integração — tendo uma visão do todo sem necessariamente ser especialista em cada parte.

Foco Prático: O objetivo é auxiliar os colaboradores a entenderem como aplicar o conhecimento dentro do contexto do banco, criando ferramentas que atendam as necessidades reais do negócio.

2. Fundamentos da Web (Request e Response)

Comunicação: A web funciona como uma conversa entre um cliente (navegador ou aplicativo) e um servidor.

Protocolo HTTP: É a linguagem dessa conversa. O HTTPS adiciona uma camada de segurança via criptografia e certificados SSL/TLS.

Infraestrutura: A configuração de servidores (Apache), sistemas operacionais e certificados é cuidada pela área de infraestrutura (DTEC), permitindo que os desenvolvedores foquem no código.

3. Anatomia de uma Requisição

Métodos Principais:

  • GET: Busca dados no servidor (ex: puxar um extrato de cliente). Os parâmetros ficam visíveis na URL.
  • POST: Envia ou grava dados (ex: preencher formulários de balanço). É mais seguro que o GET para informações sensíveis.

Estrutura da Requisição:

  • Headers (Cabeçalhos): Contêm metadados como tipo de conteúdo e encoding (ex: UTF-8).
  • Body (Corpo): Contém os dados reais, geralmente formatados em JSON (pares de chave e valor).

Status Codes:

CódigoSignificadoExemplo
200SucessoPágina carregada corretamente
404Não encontradoURL digitada não existe no servidor
500Erro internoFalha no código do servidor

Objetivo da Atividade

Desenvolver uma página web interativa simples utilizando exclusivamente tecnologias de front-end — HTML, CSS e JavaScript — sem necessidade de servidor ou banco de dados. O tema era livre: lista de tarefas, filmes favoritos, tópicos de estudo, etc.

Funcionalidades Esperadas

HTML Estruturar a página com títulos, parágrafos, listas e links.
CSS Aplicar estilos: fontes, cores de fundo e organização visual.
JavaScript Campo de texto para digitar uma informação e botão para adicioná-la a uma lista exibida na tela.

Requisito Técnico

  • A página deve ser testada localmente no navegador, sem servidor.
  • Estrutura de arquivos obrigatória: index.html, style.css e script.js.

  padding: 0; /* Remove espaço interno padrão de todos os elementos */
  box-sizing: border-box;

  border-box   padding e borda são INCLUÍDOS no tamanho declarado.

  // querySelector   igual ao querySelectorAll, mas retorna apenas
  // o PRIMEIRO elemento encontrado (ou null se não existir).
  const jaTemVazio = lista.querySelector('.vazio');
  // li:not(.vazio)   seleciona <li> que NÃO possuem a classe "vazio".
  // Isso garante que a mensagem de vazio em si não seja contada.
  if (lista.querySelectorAll('li:not(.vazio)').length === 0) {
    // if   estrutura condicional: executa o bloco {} apenas se
    // a condição entre parênteses for verdadeira.
    // ===   comparação estrita (valor E tipo devem ser iguais).
    if (!jaTemVazio) {
      // !   operador de negação. !jaTemVazio = "se NÃO existe o elemento vazio"
      // Evita adicionar a mensagem duplicada.
      const li = document.createElement('li');
      // createElement   cria um novo elemento HTML na memória.
      // O elemento ainda não está na página; precisamos inseri-lo.
      li.classList.add('vazio');
      // classList   lista de classes CSS do elemento.
      // .add('vazio')   adiciona a classe CSS "vazio" ao elemento.
      li.textContent = 'Nenhum filme na lista ainda...';
      lista.appendChild(li);
      // appendChild   insere o elemento como último filho do pai.
      // Aqui adiciona o <li> dentro da <ul id="listaFilmes">.
    }
  }

function adicionarFilme() {
  // Lê o valor atual do campo de texto
  const nome = input.value.trim();
  // .value    texto digitado pelo usuário no input.
  // .trim()   remove espaços em branco do início e fim.
  // Evita que o usuário adicione um filme só com espaços.

  // Validação: se o campo estiver vazio, interrompe a função
  if (nome === '') {
    input.focus();
    // .focus()   coloca o cursor de volta no campo, orientando o usuário.
    return;
    // return   interrompe a execução da função aqui.
    // Tudo abaixo do return não será executado neste caso.
  }

  // --- Criando o elemento <li> ---
  const li = document.createElement('li');
  // Cria o container de cada filme na lista.

  // --- Criando o <span> com o nome do filme ---
  const spanNome = document.createElement('span');
  spanNome.classList.add('nome');
  spanNome.textContent = nome; // Insere o texto digitado pelo usuário
  spanNome.title = 'Clique para marcar como assistido';
  // .title   texto que aparece como tooltip ao passar o mouse.

  // Adiciona evento de clique no nome para marcar/desmarcar como assistido

  });

  const divAcoes = document.createElement('div');

  btnCheck.classList.add('btn-check');
  btnCheck.textContent = '✓  Assistido';
